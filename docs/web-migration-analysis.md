## 3. Phase 4 任务拆解 (Web界面)

### T15: Web服务器迁移

| 属性 | 内容 |
|------|------|
| **任务ID** | T15 |
| **描述** | 迁移Bun.serve Web服务器至claude-pty |
| **风险等级** | 🟡 中 |
| **预估工时** | 8h |
| **依赖** | PTY核心模块完成 |

#### 工作内容
1. **迁移server.ts** (3h)
   - Bun.serve 配置迁移
   - 动态端口分配逻辑
   - 路由注册机制适配

2. **迁移callback-manager.ts** (2h)
   - 回调注册/注销机制
   - WebSocket广播逻辑
   - 会话事件订阅

3. **迁移handlers** (3h)
   - websocket.ts: 消息协议适配
   - sessions.ts: REST API端点
   - static.ts: 静态资源嵌入
   - health.ts: 健康检查

#### 风险点
- 🟡 **Bun.serve兼容性**: Claude Code插件环境可能限制网络端口绑定
- 🟡 **端口冲突**: 动态端口0可能受限
- 🟢 **回调生命周期**: 需确保插件卸载时正确清理

## 7. 建议

### 7.1 Phase 4 策略
- **Web界面可选**: 如时间紧张，可延后实现
- **渐进式迁移**: 先实现REST API，再实现WebSocket
- **端口绑定**: 考虑使用固定端口范围+重试机制

### 7.2 Phase 5 策略
- **先单元后集成**: 优先保证单元测试覆盖
- **E2E精选**: 选择2-3个核心场景做E2E，不追求全覆盖
- **CI适配**: 单元/集成测试在CI运行，E2E可选本地运行
